nvm 설치 문제 해결 가이드

🚨 `nvm: command not found` 오류가 발생하나요?

가장 먼저, 모든 터미널(PowerShell, cmd, Terminal 등)을 완전히 닫고 새로 열어서 명령어를 다시 입력해보세요. 대부분의 설치 문제는 터미널 재시작으로 해결됩니다.

🖥️ Windows 사용자 문제 해결

문제 1: 스크립트 실행 권한 오류

PowerShell에서 nvm 사용 시 빨간색 오류 메시지가 나타난다면, 실행 정책 문제일 수 있습니다.

해결 방법:

  1. PowerShell을 관리자 권한으로 엽니다.
  2. 아래 명령어를 입력하고 Enter를 누릅니다.
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

[Y] 예를 선택하여 정책을 변경합니다. 이제 터미널을 다시 시작하고 nvm --version을 실행해보세요.

문제 2: `nvm` 명령어를 찾을 수 없음

터미널을 재시작해도 문제가 해결되지 않는다면, 환경 변수 설정이 누락되었을 수 있습니다.

해결 방법:

nvm-windows를 재설치 해보세요. 설치 과정에서 시스템 환경 변수를 자동으로 설정해줍니다. 설치 경로에 한글이나 공백이 포함되지 않도록 주의하세요. (예: C:\nvm)

🍏 Mac/Linux 사용자 문제 해결

문제: `nvm` 명령어를 찾을 수 없음

설치 스크립트 실행 마지막에 나타나는 환경변수 설정 안내를 놓친 경우가 많습니다.

해결 방법:

사용하는 셸의 설정 파일(~/.zshrc, ~/.bash_profile 등)을 열고, 파일 맨 아래에 다음 내용을 추가하세요.

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion" # This loads nvm bash_completion

파일을 저장한 뒤, 터미널에 source ~/.zshrc (또는 사용하는 셸 파일명)를 입력하거나 터미널을 재시작하세요.

위 방법으로도 해결되지 않으면 디스코드 채널에 질문을 남겨주세요!
